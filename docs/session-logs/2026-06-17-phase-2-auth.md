## Key Decisions

| Decision | Reason |
|---|---|
| Next 16 `proxy.ts` (not `middleware.ts`) | Next 16 breaking change; confirmed from Next docs in node_modules |
| `getUser()` not `getSession()` on server | Supabase recommends `getUser()` — always server-verified, not trust-cookie |
| Pure logic in `routing.ts` / `select.ts` | Testable without mocking Next.js or Supabase; avoids server-only import boundary in tests |
| `Box` not `Stack` in AppShell toolbar | MUI v9 TypeScript overload on `Stack alignItems` prop; `Box sx` avoids the issue |
| Separate migration 0003 for anon revoke | `revoke from public` doesn't cover Supabase's direct `anon` role grants — needed explicit revoke |
| 3 remaining security advisor warnings accepted | `authenticated` calling `is_org_member`/`has_org_role`/`is_org_member_path` is intentional — they only return boolean about the caller's own membership |

## Security Posture After Phase 2

- All app routes behind auth gate (proxy.ts)
- RLS deny-by-default on `organizations` and `organization_users`
- Storage bucket `documents` private, org-namespaced, RLS-enforced
- SECURITY DEFINER helpers: `anon` execute revoked, `authenticated` only
- Open-redirect prevention in `safeRedirectPath` (double-slash, backslash, external URL)
- httpOnly cookies for org preference (not accessible from JS)
- Magic-link only (no password surface)
